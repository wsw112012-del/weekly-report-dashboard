# Claude 행동 규칙

## 주간보고 작성

"주간보고 작성해줘" 키워드가 입력되면 아래 절차를 따른다.

1. "네 알겠습니다. 유형을 선택해주세요: **데이터(AI)** 또는 **페이먼트(전자금융)**. 보도자료도 함께 첨부해주세요."라고 답변한다.
2. 사용자가 유형을 선택하고 보도자료(PDF)를 첨부하면:
   - 메모리(`feedback_weekly_report_format.md`)에 저장된 양식을 기준으로 주간보고 내용을 작성한다.
   - 첨부된 PDF의 파일명을 그대로 스크립트에 전달한다. 스크립트가 Desktop·다운로드 폴더에서 자동 탐색한다.
   - 실행 명령: `python make_report.py 데이터 "파일명.pdf"` 또는 `python make_report.py 페이먼트 "파일명.pdf"`

### 작성 형식

```
◆ 기관명 | 「정책/사안명」  '26.4.24(금)
  - 한 줄 요약

    • 주요 내용
      - 세부 내용

    • (필요 시 추가 소제목)
      - 세부 내용

    • 향후 방향 (해당 시)
      - 세부 내용
```

### 작성 원칙

- 기대효과·배경 설명 등 부연 내용은 생략
- 정책 변경 사항과 핵심 조건 위주로 작성
- PPT 1장 분량에 맞게 간결하게 유지
- 소제목은 "변경 내용" 대신 "주요 내용"으로 표기

### 세부 작성 규칙 (검증 결과 반영)

**◆ 제목 줄**
- 정책/사안명은 반드시 「」 꺾쇠 괄호로 감싼다. 예: `「개인정보 처리방침 작성지침」 개정`
- 보도자료 원문 제목을 그대로 붙여넣지 않는다. 핵심 명칭으로 축약한다.
- 말줄임표(`...`) 사용 금지 — 완결된 명칭으로 기재한다.
- 날짜 요일은 반드시 정확히 계산해서 기재한다. 예: `'26.4.24(목)` (금요일이 아닌 목요일)

**- 한 줄 요약**
- 보도자료 제목·부제목을 그대로 복사하지 않는다.
- 정책 변경의 핵심을 개조식 1줄로 직접 요약한다. 예: `- 처리방침 작성지침 개정 — 생성형 AI 서비스 기준 신설`

**• 주요 내용 세부 항목**
- 한 줄 요약과 동일한 내용을 반복하지 않는다.
- 구체적 변경 사항(신설 항목, 시행일, 적용 대상 등)을 개별 항목으로 나열한다.
- 보도자료 원문 내용을 참고해 실질적 내용을 채운다. 내용이 부족하면 원문 PDF를 첨부해 전문을 확인한다.

**보도자료 선별**
- PPT 1장 분량 내에서 복수 건 기재가 가능하므로, 주요 건이 여러 개면 함께 포함한다.

## 보도자료 수집 후 작성 (웹 UI)

"보도자료 수집해줘" 또는 "주간보고 수집해줘" 키워드가 입력되면 아래 절차를 따른다.

1. "네 알겠습니다. 유형을 선택해주세요: **데이터(AI)**, **페이먼트(전자금융)**, **AML(자금세탁방지)**, 또는 **전부 다**"라고 답변한다.
2. 사용자가 유형을 선택하면 아래 명령을 실행한다. 전부 다 선택 시 3개를 순서대로 실행한다.

```
python app.py 데이터
python app.py 페이먼트
python app.py AML
```

3. 브라우저(http://localhost:8765)가 자동으로 열리며 수집된 보도자료가 카드 형태로 표시된다.

### 웹 UI 사용 흐름

1. 브라우저에서 원하는 보도자료 카드를 클릭해 선택 (체크박스)
2. **② 선택 항목 형식화** 버튼 클릭 → ◆/•/- 형식 초안 자동 생성
3. 내용을 직접 편집 (수정 가능)
4. **③ PPT 생성** 버튼 클릭 → PPT 저장 경로 표시

## 형식화·저장 규칙

### 형식화 시 `- -` 중복 방지
- bullet 세부 항목은 반드시 6칸 들여쓰기 포함: `      - 내용`
- `_strip_prefix`로 내용 앞의 `-`, `·`, `•` 기호를 제거한 뒤 출력
- 결과 예시:
  ```
      • 주요 내용
        - 세부 내용  ✓
  -   - 세부 내용  ✗ (중복)
  ```

### PPT 저장 파일명
- 형식: `보도자료명_{날짜}.pptx`
- 보도자료명은 `◆` 줄의 「」 안 핵심 제목 추출 (특수문자 제거, 최대 40자)
- 예: `AI 기본법_20260424.pptx`

### 형식화 내용 잘림 검증
- 형식화 결과를 생성한 후 원본 제목·내용과 비교하여 핵심 내용이 누락되지 않았는지 확인
- 90자 등 하드 컷 없이 전체 내용 유지 (PPT 텍스트박스 word_wrap=True로 자동 줄바꿈)
